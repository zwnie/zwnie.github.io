直连

在建立了笔记本与虚拟机的虚拟局域网后，发现连接方式是RELAY而非DIRECT，只能经由国外中转，导致网络延迟很高。链接里提供了几种解决思路。

我搞了半天，最终没能给虚拟机打开ipv6从而直连；但我发现一个好方法，可以手机v6直连win10，再用win10直连虚拟机，达到同样的效果。如此就能随时随地办公了！

下载文献

要用到局域网内其它机器的网络环境，思路是把那台机器作为服务器，打开一个端口，其它机器就可以通过端口接入内部了。

首先，windows下载个v2ray的server。参照链接，附上服务端设置：

{
  "inbounds": [
    {
      "port": xxx, // Vmess 协议服务器监听端口
      "protocol": "vmess",
      "settings": {
        "clients": [
          {
            "id": "xxx" // id(UUID) 需要修改
          }
        ]
      }
    }
  ],
  "outbounds": [
    {
      "protocol": "freedom",
      "settings": {}
    }
  ]
}

如此一来，只要在win10-iop上打开服务，外部就能通过局域网地址和port经由主机代理访问其它网站。

需要用主机代理时，设主机为活动服务器。我们要下载文章的网站是墙内的，所以autoswitch会自动切换直连（也即手机热点），下载不了文章。用proxy强制走代理，就能通过主机进行下载了。

主机端记得开机启动。

samba

参考wiki以及网页，利用我们搭建的虚拟局域网，共享了samba文件系统的网盘。

这是我添加的内容

[smbshare]
    path = /home/niccol/smbshare
    browseable = yes
    writable = yes
    available = yes
    create mask = 777
    directory mask = 777
    valid users = niccol
    force create mode = 0644
    force directory mode = 0644
    public = yes
    case sensitive = no
    short preserve case = yes
    sync always = yes
